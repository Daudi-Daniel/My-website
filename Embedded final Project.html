Smart Water Distribution Balancer (SWDB)

Problem It Solves

Cities like Nairobi face unpredictable water supply to certain neighborhoods, leakages and pressure drops in old pipelines, and poor data on demand and waste detection. There is a need to monitor, balance, and automate water flow control based on real-time demand and pipeline status.

Project Summary

The SWDB project develops a distributed, embedded Smart Water Monitoring Node that:

  • Measures water pressure and flow rate in various pipe sections.
  • Detects leaks or bursts by analyzing pressure drops.
  • Uses data to optimize valve control for even supply based on local demand.
  • Communicates data wirelessly to a central dashboard or water utility server.


How It Works

  • Install sensor nodes at various junctions or critical points in the pipeline.
  • Each node monitors real-time flow rate and pressure before and after.
  • Detects sudden pressure drops (leaks) or no-flow conditions (blockage).
  • Sends data to a central server/dashboard.
  • Optional: Uses data to trigger flow redistribution for balanced supply.

Impact

  • Improves water supply reliability across Nairobi zones.
  • Enables early leak/burst detection, reducing water loss.
  • Supports smart urban planning and water conservation.
  • Sets foundation for Smart City infrastructure.

Future Expansion

  • AI-based demand prediction using historical data.
  • Automated pump control based on tank level and area demand.
  • Link with consumer meters for end-to-end flow accountability.

Hardware Components

  • Microcontroller Unit: ESP32 (WiFi + BLE) for urban areas; STM32 + LoRa or Arduino + GSM for remote.
  • Flow Sensor: YF-S201 or industrial turbine flow sensor (3/4" or 1").
  • Pressure Sensor: MPX5010 (low-cost) or Honeywell PX2 (industrial).
  • Optional: Water level sensor for tank monitoring.
  • Actuator: Motorized ball valve (DC 12V/24V) or solenoid valve, controlled via relay/motor driver.
  • Power: Solar panel + 18650 battery + charge controller for remote; AC adapter + backup battery for urban.
  • Communication: WiFi for urban, LoRa for long-range, GSM (SIM800/900) for backup.

Software Design

  • Embedded Code (ESP32):
    • Read pressure and flow sensor data every X seconds.
    • Calculate flow rate (L/min), pressure differential, and detect anomalies (leak = pressure drop + flow).
    • Send data over MQTT or HTTP to a central server.
    • Receive commands to open/close valve if enabled.
  • Server/Dashboard:
    • Node-RED, InfluxDB + Grafana, or Firebase for data visualization.
    • Map-based interface with real-time flow and pressure.
    • Leak alerts and manual/automatic valve control.
    • Historical data for predictive analysis.

Prototype Development Roadmap

  • Phase 1: Core Prototype
    • Breadboard setup with ESP32, 1 flow sensor, 2 pressure sensors (inlet/outlet).
    • Serial monitor/display to view data.
  • Phase 2: Communication
    • Send sensor data to a local server (Node-RED dashboard).
    • Integrate simple valve control (toggle with button or command).
  • Phase 3: Automation Logic
    • Implement leak detection logic.
    • Auto-close valve on leak or high-pressure anomaly.
  • Phase 4: Field Test Unit
    • Build waterproof node (PVC enclosure).
    • Add solar or mains power.
    • Install in small test setup (home pipeline, water tank system, etc.).
  • Phase 5: Dashboard and Expansion
    • Deploy multiple nodes and monitor flow/pressure in several branches.
    • Use map overlay for municipal setup.

Real-World Integration Ideas

  • Partner with Nairobi City Water and Sewerage Company for pilot deployment.
  • Use for community tanks, informal settlement distribution, or new housing developments.
  • Extend to manage pump stations or valve zones across estates.
